## Research Plan: Optimizing the NCADEMI Static Artifact Viewer

### Objective

To identify, validate, and document frontend and architectural best practices for building out the **NCADEMI Static Artifact Viewer**. The plan focuses on resolving the conflict between rendering full-fidelity live previews and exporting clean, semantic HTML for seamless WordPress integration, while incorporating secure, low-latency real-time stream syncs.

---

## 1. Core Research Pillars & Hypotheses

### Pillar A: "Full Preview, Partial Extraction" Isolation Mechanics

* **Context:** The viewer must load a full HTML document (including stylesheets) inside an isolated `<iframe>` using the `srcDoc` property to ensure 100% visual fidelity to the live site. However, the WordPress editor requires only inner semantic elements (`<h2>`, `<p>`, `<ul>`).


* **Hypothesis:** Utilizing the native browser `DOMParser` API on the client side allows the frontend to serve a single full-fidelity source of truth in the iframe while dynamically filtering out shell tags right before copying to the clipboard.


* **Key Question:** What fallback behaviors are necessary if a specific target class (e.g., `.entry-content`) is missing from the artifact structure?



### Pillar B: Real-Time Stream Synchronization

* **Context:** The viewer needs to reflect live research drafts and asynchronous processing statuses in real time[cite: 1, 5, 6].
* **Hypothesis:** Utilizing a userland stream consumer like `@microsoft/fetch-event-source` provides superior header manipulation (handling Firebase ID tokens) over the native `EventSource` API, minimizing backend state lookups and optimization blockers.


* **Key Question:** How does token expiration (1-hour lifecycle) affect long-lived viewer connections, and can an `onopen` interceptor elegantly resolve re-authentication loops?

## 4. Risks and Mitigation Strategies

* **Risk 1: Third-Party Cookie Blocks on Default Domains.** Attempting to use ambient credentials (cookies) across disparate `.run.app` or `.web.app` origins triggers strict browser privacy flags due to the Public Suffix List.


* *Mitigation:* Enforce authorization entirely via explicitly managed HTTP Headers in `fetch-event-source`, bypassing third-party cookie filters without requiring a Global External Load Balancer.




* **Risk 2: Phantom Reconnection Storms.** If a mobile device backgrounds the browser tab, suspended JavaScript execution loop handlers can trigger heavy rate-limiting errors (`429 Too Many Requests`).


* *Mitigation:* Utilize Page Visibility API hooks within the viewer shell to gracefully drop connections when backgrounded and refresh credentials only upon view resumption.

### 1. Real-Time Stream Synchronization Hook (`useAuthenticatedSSE.ts`)

This custom React hook replaces the native `EventSource` API with `@microsoft/fetch-event-source` to enable custom header injection for Firebase Identity tokens. It specifically addresses the one-hour token expiration lifecycle by implementing an `onopen` interceptor that catches HTTP 401 Unauthorized responses, cleanly aborts the connection, mints a fresh token via the client SDK, and seamlessly reconnects without exposing the drop to the UI.

```typescript
import { useState, useEffect, useRef } from 'react';
import { fetchEventSource } from '@microsoft/fetch-event-source';
import { getAuth, getIdToken } from 'firebase/auth';

interface SSEOptions {
  url: string;
  isEnabled?: boolean;
}

export const useAuthenticatedSSE = ({ url, isEnabled = true }: SSEOptions) => {
  const [streamData, setStreamData] = useState<string>('');
  const [status, setStatus] = useState<'idle' | 'connecting' | 'connected' | 'error'>('idle');
  const abortCtrl = useRef<AbortController | null>(null);

  useEffect(() => {
    if (!isEnabled) {
      setStatus('idle');
      return;
    }

    const connect = async () => {
      setStatus('connecting');
      abortCtrl.current = new AbortController();
      const auth = getAuth();
      const user = auth.currentUser;

      try {
        // Fetch standard cached token; fallback to empty string if unauthenticated
        let token = user ? await getIdToken(user, false) : '';

        await fetchEventSource(url, {
          method: 'GET',
          headers: {
            Authorization: `Bearer ${token}`,
            Accept: 'text/event-stream',
          },
          signal: abortCtrl.current.signal,
          
          async onopen(response) {
            // Intercept token expiration before the stream parses
            if (response.status === 401) {
              if (user) {
                // Force a token refresh
                await getIdToken(user, true);
                throw new Error('401_UNAUTHORIZED'); 
              }
            }
            if (response.ok) {
              setStatus('connected');
              return;
            }
            throw new Error(`Server responded with ${response.status}`);
          },
          
          onmessage(msg) {
            // Append incoming text chunks
            setStreamData((prev) => prev + msg.data);
          },
          
          onclose() {
            setStatus('idle');
          },
          
          onerror(err) {
            if (err.message === '401_UNAUTHORIZED') {
              // The outer catch block will handle the reconnect
              throw err; 
            }
            // Stop retrying on other fatal infrastructure errors
            throw err; 
          }
        });
      } catch (error: any) {
        if (error.message === '401_UNAUTHORIZED') {
          // Abort the dead connection and immediately cycle a fresh one
          abortCtrl.current?.abort();
          connect(); 
        } else {
          console.error("SSE Connection Error:", error);
          setStatus('error');
        }
      }
    };

    connect();

    // Cleanup: gracefully terminate TCP connection on unmount or URL change
    return () => {
      abortCtrl.current?.abort();
    };
  }, [url, isEnabled]);

  return { streamData, status };
};

```

### 2. WordPress Extraction Utility (`wordpressExtractor.ts`)

This utility executes the "Full Preview, Partial Extraction" paradigm. It utilizes the native `DOMParser` to parse the complete HTML string generated by the Fast API backend, searches for the targeted `.entry-content` wrapper, and copies only the pure semantic inner content to the clipboard, allowing seamless integration with the Gutenberg block editor.

```typescript
/**
 * Extracts clean, semantic HTML from a full HTML document string
 * and copies it to the user's clipboard.
 * * @param {string} fullHtmlString - The complete HTML document returned by the API
 */
export const copyWordpressReadyHtml = async (fullHtmlString: string): Promise<void> => {
  try {
    // 1. Parse the raw HTML string into a queryable DOM object
    const parser = new DOMParser();
    const doc = parser.parseFromString(fullHtmlString, 'text/html');

    // 2. Target the specific wrapper used by the NCADEMI WordPress theme.
    // Fallback to <main> or <body> if the specific class isn't found.
    const targetNode = doc.querySelector('.entry-content') 
                    || doc.querySelector('main') 
                    || doc.body;

    if (!targetNode) {
      throw new Error("Could not locate the primary content container.");
    }

    // 3. Extract ONLY the inner HTML (semantic content, no shell tags)
    const cleanHtml = targetNode.innerHTML.trim();

    // 4. Write to the clipboard securely
    if (navigator.clipboard && window.isSecureContext) {
      await navigator.clipboard.writeText(cleanHtml);
    } else {
      fallbackCopyTextToClipboard(cleanHtml);
    }
    
    alert("WordPress-ready HTML copied to clipboard!");

  } catch (error) {
    console.error("Failed to extract and copy HTML:", error);
    alert("Failed to extract HTML. Check console for details.");
  }
};

/**
 * Fallback for older browsers or insecure iframe contexts
 * using the legacy document.execCommand
 */
const fallbackCopyTextToClipboard = (text: string) => {
  const textArea = document.createElement("textarea");
  textArea.value = text;
  
  // Prevent scrolling to bottom of page in MS Edge
  textArea.style.position = "fixed";
  textArea.style.top = "0";
  textArea.style.left = "0";
  textArea.style.width = "2em";
  textArea.style.height = "2em";
  textArea.style.padding = "0";
  textArea.style.border = "none";
  textArea.style.outline = "none";
  textArea.style.boxShadow = "none";
  textArea.style.background = "transparent";
  
  document.body.appendChild(textArea);
  textArea.focus();
  textArea.select();
  
  try {
    document.execCommand('copy');
  } catch (err) {
    console.error('Fallback: Oops, unable to copy', err);
  }
  
  document.body.removeChild(textArea);
};

```

# CLOUD TASKS OIDC ENQUEUER FOR PRIVATE CLOUD RUN WORKERS
# ============================================================================

def enqueue_secure_worker_task(
    project_id: str,
    location: str,
    queue_id: str,
    target_worker_url: str,
    invoker_service_account_email: str,
    payload: dict,
    task_name: str = None
) -> tasks_v2.Task:
    """
    Constructs and enqueues an HTTP Cloud Task with an OIDC token to invoke 
    a private Cloud Run worker service.
    
    Adheres strictly to the "Audience Mismatch" and "Trailing Slash" rules 
    identified in the Cloud Run Authentication Research.
    """
    
    # Initialize the Cloud Tasks Client
    client = tasks_v2.CloudTasksClient()

    # Construct the fully qualified queue name
    parent = client.queue_path(project_id, location, queue_id)

    # Convert the dictionary payload to a JSON string, then encode to bytes
    encoded_payload = json.dumps(payload).encode()

    # CRITICAL: Prevent OIDC Audience Mismatches & Trailing Slash Vulnerabilities
    # The audience MUST perfectly match the root .run.app URL without a trailing slash.
    parsed_url = urlparse(target_worker_url)
    clean_audience = f"{parsed_url.scheme}://{parsed_url.netloc}"

    # Construct the base task definition
    task = {
        "http_request": {
            "http_method": tasks_v2.HttpMethod.POST,
            "url": target_worker_url,
            "headers": {"Content-type": "application/json"},
            "body": encoded_payload,
            
            # CRITICAL: OIDC Authorization Block
            # This instructs the Cloud Tasks infrastructure to mint an identity token
            "oidc_token": {
                "service_account_email": invoker_service_account_email,
                "audience": clean_audience
            },
        }
    }

    # Optional: Implement idempotency by specifying a specific task name
    if task_name:
        # Task names must be fully qualified
        task["name"] = client.task_path(project_id, location, queue_id, task_name)

    try:
        # Enqueue the task
        # NOTE: The API service account running this code must have:
        # 1. roles/cloudtasks.enqueuer on the queue
        # 2. roles/iam.serviceAccountUser on the 'invoker_service_account_email'
        response = client.create_task(request={"parent": parent, "task": task})
        print(f"Successfully enqueued task: {response.name}")
        return response
        
    except Exception as e:
        print(f"Failed to enqueue secure task: {str(e)}")
        raise

import os
import json
import logging
from fastapi import FastAPI, Header, HTTPException, Request
from pydantic import BaseModel
import google.generativeai as genai

# Configure logging for Cloud Logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

app = FastAPI(title="NCADEMI Artifact Worker")

# Initialize Gemini API (Expects GEMINI_API_KEY environment variable)
# Using the standard gemini-1.5-pro or flash model for document processing
genai.configure(api_key=os.environ.get("GEMINI_API_KEY"))
model = genai.GenerativeModel('gemini-1.5-flash')

class ArtifactPayload(BaseModel):
    product_slug: str
    target_url: str
    current_html: str

@app.post("/process-artifact")
async def process_artifact(
    payload: ArtifactPayload,
    request: Request,
    x_cloudtasks_taskname: str = Header(None),
    x_cloudtasks_taskretrycount: int = Header(0)
):
    """
    Secure endpoint invoked by Cloud Tasks to process a website artifact
    using the Gemini API and generate a N.E.R.D draft update.
    """
    
    # 1. Zero-Trust Verification
    # While IAM handles authentication, we ensure this route is ONLY hit by Cloud Tasks
    # in a production environment by checking the presence of the system header.
    is_prod = os.environ.get("ENVIRONMENT") == "production"
    if is_prod and not x_cloudtasks_taskname:
        logger.error("Unauthorized invocation attempt: Missing Cloud Tasks headers.")
        raise HTTPException(status_code=403, detail="Direct invocation forbidden.")

    logger.info(f"Received task: {x_cloudtasks_taskname} for slug: {payload.product_slug}")
    logger.info(f"Retry count: {x_cloudtasks_taskretrycount}")

    # 2. Dead-Letter Queue Logic
    if x_cloudtasks_taskretrycount > 3:
        logger.error(f"Task {x_cloudtasks_taskname} exceeded retry limits. Shunting to DLQ.")
        # In a real app, write to a Firestore DLQ collection here
        return {"status": "shunted_to_dlq"}

    try:
        # 3. Gemini Processing Pipeline
        prompt = f"""
        Analyze the following HTML content for the product '{payload.product_slug}'.
        Generate an updated 'N.E.R.D.' (NCADEMI Educational Resource Database) accessibility 
        draft in semantic HTML. 
        
        Rules:
        1. Keep the output strictly within a <div class="entry-content"> wrapper.
        2. Extract and preserve any existing vendor support links.
        3. Add an <div class="ai-insights"> section summarizing the accessibility posture based on modern WCAG 2.1 AA standards.
        
        Raw HTML to analyze:
        {payload.current_html}
        """

        logger.info("Invoking Gemini API...")
        response = model.generate_content(prompt)
        
        updated_html = response.text

        # 4. Save to Database (Mocked)
        # Here you would use the Firebase Admin SDK to update the document,
        # which subsequently triggers the Frontend SSE connection.
        # db.collection('artifacts').document(payload.product_slug).update({
        #     'updatedHtml': updated_html,
        #     'metadata.nerd_status': 'draft_complete'
        # })

        logger.info(f"Successfully processed artifact for {payload.product_slug}")
        
        return {
            "status": "success", 
            "product": payload.product_slug, 
            "bytes_generated": len(updated_html)
        }

    except Exception as e:
        logger.error(f"Error processing task: {str(e)}")
        # Raising a 500 signals Cloud Tasks to retry the payload with exponential backoff
        raise HTTPException(status_code=500, detail="Internal processing error")

if __name__ == "__main__":

def cmd_enqueue(args):
    """Command to manually dispatch a Cloud Task to the worker."""
    if not all([args.project, args.location, args.queue, args.worker_url, args.service_account]):
        print("Error: Missing required configuration arguments for enqueuing.")
        sys.exit(1)

    print(f"Reading payload from {args.payload_file}...")
    try:
        with open(args.payload_file, 'r') as f:
            payload = json.load(f)
    except Exception as e:
        print(f"Failed to load payload: {e}")
        sys.exit(1)

    print(f"Dispatching task for product: {payload.get('product_slug', 'unknown')}")
    
    try:
        response = enqueue_secure_worker_task(
            project_id=args.project,
            location=args.location,
            queue_id=args.queue,
            target_worker_url=args.worker_url,
            invoker_service_account_email=args.service_account,
            payload=payload
        )
        print(f"\nSuccess! Task ID: {response.name}")
    except Exception as e:
        print(f"\nFailed to enqueue task: {e}")
        sys.exit(1)

def cmd_analyze_local(args):
    """Command to bypass Cloud Tasks and test the Gemini extraction locally."""
    import google.generativeai as genai
    
    api_key = os.environ.get("GEMINI_API_KEY")
    if not api_key:
        print("Error: GEMINI_API_KEY environment variable is missing.")
        sys.exit(1)
        
    print(f"Reading target HTML from {args.html_file}...")
    try:
        with open(args.html_file, 'r') as f:
            raw_html = f.read()
    except Exception as e:
        print(f"Failed to load HTML: {e}")
        sys.exit(1)

    print("Initializing local Gemini Model (gemini-1.5-flash)...")
    genai.configure(api_key=api_key)
    model = genai.GenerativeModel('gemini-1.5-flash')
    
    prompt = f"""
    Analyze this HTML layout. Extract ONLY the semantic core content (headings, paragraphs, lists)
    and wrap it in <div class="entry-content">. Discard navigation, footers, and scripts.
    
    HTML:
    {raw_html}
    """
    
    print("Generating N.E.R.D draft...\n")
    response = model.generate_content(prompt)
    
    print("--- RESULT ---")
    print(response.text)
    print("--------------")
    
    if args.output:
        with open(args.output, 'w') as f:
            f.write(response.text)
        print(f"Result saved to {args.output}")

    parser = argparse.ArgumentParser(description="Gemini CLI Agent for NCADEMI Artifact Processing")

Phase 5: Worker Service Deployment & Validation

The worker_service.py acts as the secure terminus for your Cloud Tasks queue. It processes the HTML using the Gemini API.

1. Deploy the Worker to Cloud Run

Ensure you deploy the service with the --no-allow-unauthenticated flag to enforce the Zero-Trust configuration established in Phase 4.

gcloud run deploy ncademi-artifact-worker \
  --source . \
  --region us-central1 \
  --no-allow-unauthenticated \
  --ingress internal-and-cloud-load-balancing \
  --service-account YOUR_WORKER_SA@YOUR_PROJECT_ID.iam.gserviceaccount.com \
  --set-env-vars="ENVIRONMENT=production,GEMINI_API_KEY=your_secure_api_key"


2. Verify Idempotency and Headers

As established in the architectural plan, the worker reads X-CloudTasks-TaskName. Because Cloud Tasks guarantees "at least once" delivery, network partitions can cause a task to be delivered twice.

Best Practice: Use the X-CloudTasks-TaskName as a Document ID in a Firebase/Firestore transactions collection. If the document already exists, return 200 OK immediately without calling the Gemini API to save inference costs.
